##慕课网上O2O的项目知识点

  • mysql中两种时间类型DATETIME和TIMESTAMP。DATETIME的时间范围大于TIMESTAMP，但TIMESTAMP会自适应你所在的时区。
  • mysql有两种引擎InnoDB和MYISAM。
    1. MyISAM是基于表级锁的,对MyISAM表的读操作，不会阻塞其他用户对同一表的读请求，但会阻塞对同一表的写请求；对 MyISAM表的写操作，则会阻塞其他用户对同一表的读和写操作；MyISAM表的读操作与写操作之间，以及写操作之间是串行的！已被放弃。
    2. InnoDB是MySQL默认的事务型存储引擎，只有在需要 InnoDB不支持的特性时，才考虑使用其它存储引擎。提供行锁,采用 MVCC 来支持高并发，并且实现了四个标准的隔离级别，默认级别是可重复读（REPEATABLE READ）,并且通过间隙锁（next-key locking）策略防止幻读的出现。间隙锁使得 InnoDB 不仅仅锁定查询涉及的行，还会对索引中的间隙进行锁定，以防止幻影行的插入。
  • 索引就类似于我们字典中的偏旁和部首，方便我们快速查询。索引过多也会影响数据库的性能。

6.添加一个CodeUtil,用来判断验证码是否符合预期的验证码。

public class CodeUtil {
 public static boolean checkVerifyCode(HttpServletRequest request){
    String verifyCodeExpected= (String) request.getSession().getAttribute(
    com.google.code.kaptcha.Constants.KAPTCHA_SESSION_KEY);//获取图片中的验证码
        String verifyCodeActual = HttpServletRequestUtil.getString(request, "verifyCodeActual");//用户实际输入的验证码
        if(verifyCodeActual == null || !verifyCodeActual.equals(verifyCodeExpected)){
            return false;
        }
        return true;
    }
}

@RequestMapping(value="/registershop",method = RequestMethod.POST)

        modelMap.put("errMsg", "输入了错误的验证码");
        return modelMap;
    }
  • 分页的实现
    1.在shopDao层接口定义一个List方法
    /**
    • 分页查询店铺，可输入的条件有：店铺名（模糊）店铺类别，店铺状态
    • @param shopCondition 查询的条件
    • @param rowIndex 从第几行开始取数据
    • @param pageSize 返回的条数
    • @return
      */
      List queryShopList(@Param(“shopCondition”)Shop shopCondition,
      @Param(“rowIndex”) int rowIndex,@Param(“pageSize”)int pageSize);
      /**
    • 返回queryShopList总数
    • @param shopCondition
    • @return
      */
      int queryShopCount(@Param(“shopCondition”)Shop shopCondition);

这里为什么要加@Param，之前的方法中参数只有一个，去参数的值去直接取就ok了。而这里有三个参数，必须要用@param的唯一的标识，根据唯一的标识才能取到正确的值。
